# Poster Compositor — Phase 2A Spike

Proves the deterministic-compositor concept: a clean, product-anchored background
PNG + an `overlay_spec` (recipe zones) → a crisp **1080×1920** poster PNG where
text is drawn by Chromium (real fonts), not by diffusion. **Spike only** — no
service, no UI, no live generation, no credit spend, not production quality.

## Runtime availability (audit correction #1)
Verified on this Windows host before building:
- Node `v24.14.1`
- Playwright `1.60.0` (already a root devDependency — `package.json`)
- Chromium **installed**: `…/ms-playwright/chromium-1223/chrome-win64/chrome.exe` (`chromium_installed: true`)

`node scripts/poster-compositor-spike.js --probe` re-checks this and exits non-zero
if Chromium is missing. **No new renderer family is introduced; the existing
Playwright dependency is reused because runtime availability is verified.**

## What the spike does
`scripts/poster-compositor-spike.js`:
- `--probe` — runtime availability.
- `--make-sample-bg` — writes a clean, product-anchored `sample_background.png`
  (warm scene + a product placeholder **with its own label preserved** in the
  product region, and **no marketing text**).
- (default) — loads `sample_background.png` + `sample_overlay_spec.json` +
  `sample_product_safe_region.json`, renders a 1080×1920 HTML canvas
  (background `<img>` + absolutely-positioned zone `<div>`s at `%x/y/w/h`,
  `font_role` → font, alignment, wrap, clamp), screenshots to
  `spike_output.png`, measures each zone's box, and writes `rendered_zones.json`.
  It **throws** if dimensions ≠ 1080×1920, any zone overflows, or any zone
  overlaps the product region.

## Recorded proof (`rendered_zones.json`, committed)
- `output_png`: **1080×1920**
- 6 zones (headline / subheadline / chip_1..3 / cta) — all `fitted: true`,
  `overflowed: false`, `overlaps_product: false`
- `credit_spend: false`, `network: false`
- `rect_pct` matches the authored `overlay_spec` coordinates (the %→CSS mapping is exact)

(The `.png` outputs are regenerated by the script and are not committed;
`rendered_zones.json` is the durable machine-checkable proof.)

## Product-identity rule (audit correction #3)
The clean background **preserves** the real product label, logo, cap, packaging
text, bottle, and product identity (product truth). Only **generated marketing
text** (headline / subhook / USP chips / CTA / footer) is absent so the
compositor owns it. This is never a broad "no text".

## Product-safe / no-text region (audit correction #4)
`product_safe_region` (percent rect of the product hero). Invariant: **no overlay
zone may intersect it.** Enforced in the spike (throws) and asserted twice in the
contract test (from the recorded run and independently from the authored fixtures).

## Discovery gap resolution (audit correction #6)
- **Flag-level, cited:** the "clean-frame no-text" convention (`allows_rendered_text:
  False`) applies to non-poster lanes; **`PRODUCT_POSTER` is the exception** —
  `agent/services/img_asset_lane_config.py:156-157`
  (`allows_rendered_text: True`, `default_contains_rendered_text: True`) vs
  PRODUCT_HERO `:128-129` (both `False`); purpose `:137-139`
  ("*with rendered headline / copy / CTA*").
- **Remaining for Phase 2B:** the current poster background negative
  (`poster_prompt_draft_service.py` `BASE_NEGATIVE_PROMPT`) bans medical/before-after
  text but does **not** suppress marketing headline/CTA text (because the poster
  lane currently *wants* rendered text). Phase 2B must add a poster-clean negative
  clause — *"no marketing headline/USP/CTA overlay text; preserve product label,
  logo, cap, packaging text"* — so the background is clean for compositing while
  product identity is preserved.

## Test strategy applied (audit correction #5)
`tests/ui/test_poster_compositor_spike_contract.py` uses **DOM-measured zone
metadata + dimensions + fitted/overflow flags + geometric overlap checks** (not
byte-hash). Strict byte-hash golden comparison is deferred to a pinned CI/runtime
env (Phase 2D).

## Explicit limits (do not overclaim)
- **Spike only** — not a service, not wired to the UI, not the live generation lane.
- **System fonts** (Segoe UI/Arial). Noto Sans / Tamil / SC bundling for
  production script coverage is **Phase 2D**.
- **No credit spend, no live generation, no network** (asserted).
- **Not production-quality poster output. LIVE PRODUCT VISUAL MATCH NOT VERIFIED.**
- Next phases (not started): **2B** compositor service (endpoint + contracts +
